# DMD Cloud - Complete Workflow Guide

## Overview

The **DMD Cloud** project includes an automated CI/CD system that:
1. Builds Docker images for all services
2. Deploys to a single-node Minikube Kubernetes cluster
3. Sends webhook requests to trigger pipeline generation
4. Uses DeepSeek AI to generate optimized GitHub Actions workflows
5. Automatically saves the generated workflow to `.github/workflows/ci-cd.yml`

## Architecture

```
Code Push
    ↓
GitHub Webhook (optional - for testing locally)
    ↓
Webhook Service (FastAPI)
    ↓
AI Service (FastAPI) 
    ↓
DeepSeek API (Generate YAML)
    ↓
.github/workflows/ci-cd.yml (Generated Workflow)
```

## Quick Start

### Step 1: Start Minikube

```powershell
minikube start --memory=4096 --cpus=2 --driver=docker
```

Verify it's running:
```powershell
minikube status
kubectl get nodes
```

### Step 2: Run the Complete Workflow

```powershell
# Navigate to project directory
cd c:\Users\tosim\dmd-cloud-project

# Run the complete workflow
.\run-workflow.ps1 -DeepSeekAPIKey "sk-your-api-key-here"
```

**What this script does:**
1. ✓ Validates prerequisites (git, docker, kubectl, minikube)
2. ✓ Builds Docker images for all services
3. ✓ Loads images into Minikube
4. ✓ Creates Kubernetes secret with DeepSeek API key
5. ✓ Deploys AI Service and Webhook Service to Minikube
6. ✓ Sets up port forwarding (localhost:8001 for webhook, localhost:8000 for AI)
7. ✓ Tests service health checks
8. ✓ Sends a webhook request with sample code diff
9. ✓ Monitors workflow execution (60 seconds)
10. ✓ Verifies generated `.github/workflows/ci-cd.yml`

# Cleanup resources after testing

### Combined Options
```powershell

# Development flow: build, deploy, skip cleanup
.\run-workflow.ps1 -DeepSeekAPIKey "sk-xxx"

# Production simulation: full workflow with cleanup
.\run-workflow.ps1 -DeepSeekAPIKey "sk-xxx" -CleanupAfter
```

## Workflow Stages Explained

### Stage 1: Prerequisites Check (✓)
- Verifies git, docker, kubectl, minikube are installed
- Checks Minikube is running and accessible
- Validates repository structure

### Stage 2: Build Docker Images (✓)
- Builds `ai-service:latest`
  - FastAPI service for AI pipeline generation
  - Communicates with DeepSeek API
  - Sanitizes and returns generated YAML

- Builds `webhook-service:latest`
  - FastAPI service for GitHub webhooks
  - Receives code diffs and repository info
  - Calls AI service asynchronously
  - Saves generated pipelines to file system

### Stage 3: Load into Minikube (✓)
- Exports Docker images to Minikube's container runtime
- Makes images available for Kubernetes pods

### Stage 4: Create Kubernetes Secret (✓)
- Creates `ai-service-secrets` secret in Kubernetes
- Stores DeepSeek API key securely
- Mounted to AI service pods

### Stage 5: Deploy to Kubernetes (✓)
- Deploys `ai-service` (1 replica)
  - Service type: ClusterIP (internal only)
  - Port: 8000
  - Resource limits: 512Mi memory, 500m CPU

- Deploys `webhook-service` (2 replicas)
  - Service type: NodePort
  - Port: 8001
  - Health checks configured
  - Resource limits: 256Mi memory, 500m CPU

- Waits for deployments to reach "Ready" state

### Stage 6: Port Forwarding (✓)
- `localhost:8001` → `webhook-service:8001`
- `localhost:8000` → `ai-service:8000`
- Enables local testing without exposing services

### Stage 7: Health Checks (✓)
- Tests `/health` endpoint on both services
- Retries up to 10 times with 3-second intervals
- Ensures services are ready before testing

### Stage 8: Send Webhook (✓)
- Sends HTTP POST to `http://localhost:8001/webhook/github`
- Payload includes:
  - Repository name
  - Commit ID and message
  - Sample code diff (auth.py addition)
- Webhook processed asynchronously

### Stage 9: Monitor Execution (✓)
- Waits 60 seconds for AI processing
- Monitors webhook service logs
- Shows progress bar and status updates
- Displays relevant log entries

### Stage 10: Verify Generated Workflow (✓)
- Checks if `.github/workflows/ci-cd.yml` exists
- Displays first 30 lines of generated workflow
- Validates YAML structure
- Shows file size and modification time

## Generated Workflow Structure

The AI service generates a complete GitHub Actions workflow containing:

### Jobs Included
1. **code-quality** - Linting and code formatting checks
2. **build-images** - Build Docker images for all services
3. **validate-kubernetes** - Validate K8s manifests
4. **unit-tests** - Run Python tests with coverage
5. **integration-tests** - Deploy to Minikube and test
6. **security-scan** - Vulnerability scanning with Trivy
7. **generate-docs** - Auto-generate documentation
8. **workflow-status** - Final status summary

### Triggers
```yaml
on:
  push:
    branches: [master, develop]
  pull_request:
    branches: [master, develop]
  workflow_dispatch:
```

## Troubleshooting

### Issue: "Minikube is not running"
```powershell
minikube start --memory=4096 --cpus=2
```

### Issue: Port 8001 already in use
```powershell
# Kill existing port-forward jobs
Get-Job -Name "minikube-port-forward-*" | Stop-Job
Get-Job -Name "minikube-port-forward-*" | Remove-Job
```

### Issue: Webhook service not healthy
```powershell
# Check service logs
kubectl logs -l app=webhook-service --tail=20

# Check pod status
kubectl describe pod -l app=webhook-service
```

### Issue: AI service not processing
```powershell
# Verify secret exists
kubectl get secrets
kubectl get secret ai-service-secrets -o yaml

# Check AI service logs
kubectl logs -l app=ai-service --tail=20

# Test AI service directly
curl http://localhost:8000/health
```

### Issue: Generated workflow file not created
```powershell
# Check webhook logs for errors
kubectl logs -l app=webhook-service --since=2m

# Verify the webhook received the request
kubectl logs -l app=webhook-service | grep -i "Received webhook"

# Check if .github/workflows directory exists
ls -la .github/workflows/
```

### Issue: Docker images not found in Minikube
```powershell
# Re-load images into Minikube
minikube image load ai-service:latest
minikube image load webhook-service:latest

# Verify images loaded
minikube image ls | grep -E "(ai-service|webhook-service)"
```

## Advanced Operations

### Manual Kubernetes Deployment

If you prefer to skip the script:

```powershell
# 1. Create secret
kubectl create secret generic ai-service-secrets `
  --from-literal=DEEPSEEK_API_KEY="sk-xxx"

# 2. Deploy services
kubectl apply -f k8s/ai-deployment.yaml
kubectl apply -f k8s/webhook-deployment.yaml

# 3. Wait for rollout
kubectl rollout status deployment/webhook-service --timeout=120s

# 4. Port forward
kubectl port-forward service/webhook-service 8001:8001
```

### Manual Webhook Test

```powershell
$payload = @{
    repository = @{ full_name = "dmd-cloud/dmd-cloud-project" }
    head_commit = @{ 
        id = "abc123"
        message = "test commit"
    }
    diff = @"
diff --git a/test.py b/test.py
+print('Hello World')
"@
} | ConvertTo-Json -Depth 10

Invoke-WebRequest -Uri "http://localhost:8001/webhook/github" `
  -Method POST `
  -Body $payload `
  -ContentType "application/json"
```
