### Core Application Services
```
webhook_service/main.py (FastAPI)
├─ POST /webhook/github
│  ├─ Receives GitHub webhook events
│  ├─ Returns 200 OK immediately (github's timeout: 30s)
│  └─ Queues background async task
│
├─ Background async task: call_ai_async()
│  ├─ Extracts code diff from payload
│  ├─ POSTs to http://ai-service:8000/generate-pipeline
│  ├─ Implements retry logic (3x, exponential backoff)
│  ├─ Timeout: 90 seconds
│  └─ On success: save_generated_pipeline()
│
├─ save_generated_pipeline(content)
│  ├─ Creates .github/workflows/ directory
│  ├─ Writes YAML to .github/workflows/ci-cd.yml
│  └─ Logs full YAML content
│
└─ GET /health
   └─ K8s liveness/readiness probe → HTTP 200
   └─ Checks AI service health too

ai_service/main.py (FastAPI)
├─ POST /generate-pipeline
│  ├─ Receives {diff, repository, commit_message}
│  ├─ Builds prompt: "Generate GitHub Actions YAML"
│  ├─ Calls DeepSeek API (HTTPS): https://api.deepseek.com/v1/chat/completions
│  ├─ Timeout: 60 seconds
│  ├─ Model: deepseek-chat
│  ├─ Sanitizes response (removes prose, markdown)
│  └─ Returns pure YAML only
│
├─ sanitize_pipeline_yaml(content)
│  ├─ Removes markdown fences (```)
│  ├─ Finds YAML start (name:, on:, jobs:)
│  └─ Removes leading prose/explanations
│
└─ GET /health
   └─ K8s probe → HTTP 200: {"status":"ok"}
```

### Kubernetes Configuration
```
k8s/webhook-deployment.yaml
├─ Kind: Deployment
├─ Name: webhook-service
├─ Replicas: 2 (for redundancy)
├─ Image: webhook-service:autosave1
├─ Port: 8001 (NodePort)
├─ Service Type: NodePort (exposes to minikube)
├─ Environment Variables:
│  ├─ AI_SERVICE_URL=http://ai-service:8000
│  ├─ AI_SERVICE_TIMEOUT_SECONDS=90
│  ├─ AUTO_SAVE_PIPELINE=true
│  └─ PIPELINE_OUTPUT_PATH=.github/workflows/ci-cd.yml
├─ Health Checks:
│  ├─ Liveness: GET /health every 30s (restart if down)
│  └─ Readiness: GET /health every 10s (traffic if ready)
└─ Resources: CPU 250m, Memory 64Mi request / 256Mi limit

k8s/ai-deployment.yaml
├─ Kind: Deployment
├─ Name: ai-service
├─ Replicas: 1
├─ Image: ai-service:yamlfix1
├─ Port: 8000 (ClusterIP, internal only)
├─ Service Type: ClusterIP (no external access)
├─ Environment Variables:
│  └─ DEEPSEEK_TIMEOUT_SECONDS=60
├─ Secret Mount:
│  └─ DEEPSEEK_API_KEY from ai-service-secrets
├─ Health Checks:
│  ├─ Liveness: GET /health every 30s
│  └─ Readiness: GET /health every 10s
└─ Resources: CPU 250m, Memory 256Mi request / 512Mi limit

k8s/ai-service-secret.template.yaml
├─ Kind: Secret
├─ Name: ai-service-secrets
└─ Data: DEEPSEEK_API_KEY: sk-...
```

## How to Navigate This Repository

### I'm New - Where Do I Start?
1. Read [README.md](README.md) - Complete overview (10 min)
2. Run [test-workflow.ps1](test-workflow.ps1) - See it working (2 min)
3. Check pod status:
   ```bash
   kubectl get pods -A
   kubectl logs <webhook-pod-id> -f
   ```
4. Read [ARCHITECTURE.md](ARCHITECTURE.md) - Understand the system (20 min)

### I'm a Developer - What Do I Need?
1. **webhook_service/main.py** - How webhook processes events
2. **ai_service/main.py** - How AI generates pipelines
3. [README.md](README.md) - "API Reference" section for endpoints
4. [ARCHITECTURE.md](ARCHITECTURE.md) - "UML Diagrams" section
5. `test-webhook.json` - Sample payload for testing

### I'm a DevOps/SRE - What Do I Need?
1. **k8s/** - Kubernetes resource definitions
2. [README.md](README.md) - "Kubernetes Cheat Sheet" section
3. [ARCHITECTURE.md](ARCHITECTURE.md) - "Kubernetes Resource Definitions"
4. `test-workflow.ps1` - Automated testing
5. [README.md](README.md) - "Environment Variables" section

### I Need to Deploy This - What's My Checklist?
1. ✅ Read [README.md](README.md) - "5-Minute Quick Start"
2. ✅ Set DEEPSEEK_API_KEY in k8s/ai-service-secret.yaml
3. ✅ Build images:
   ```bash
   docker build -t webhook-service:autosave1 ./webhook_service
   docker build -t ai-service:yamlfix1 ./ai_service
   minikube image load webhook-service:autosave1
   minikube image load ai-service:yamlfix1
   ```
4. ✅ Deploy to Minikube:
   ```bash
   kubectl apply -f k8s/ai-service-secret.yaml
   kubectl apply -f k8s/ai-deployment.yaml
   kubectl apply -f k8s/webhook-deployment.yaml
   ```
5. ✅ Test:
   ```bash
   .\test-workflow.ps1
   ```

### I Need to Troubleshoot - What Do I Read?

## File Modification Guide

### Safe to Modify
- `webhook_service/main.py` - Add features, change logic
- `ai_service/main.py` - Adjust prompt, add features
- `k8s/*.yaml` - Scale replicas, change timeouts
- `test-workflow.ps1` - Improve testing logic
- `README.md` - Update documentation
- `ARCHITECTURE.md` - Add diagrams

### Don't Modify
- `.gitignore` - Git needs this unchanged
- `test-webhook.json` - Sample data (read-only reference)
- `terraform/main.tf` - Only if deploying to cloud

### Never Commit to Git
- `k8s/ai-service-secret.yaml` with real API key
- `.env` files with credentials
- `*.log` files

## Common Commands

### Build Images
```bash
cd dmd-cloud-project
docker build -t webhook-service:autosave1 ./webhook_service
docker build -t ai-service:yamlfix1 ./ai_service
minikube image load webhook-service:autosave1
minikube image load ai-service:yamlfix1
```

### Deploy to Minikube
```bash
kubectl apply -f k8s/ai-service-secret.yaml
kubectl apply -f k8s/ai-deployment.yaml
kubectl apply -f k8s/webhook-deployment.yaml
```

### Run Tests
```bash
.\test-workflow.ps1
```

### View Logs
```bash
kubectl logs -f deployment/webhook-service
kubectl logs -f deployment/ai-service
```

### Access Services Locally
```bash
kubectl port-forward svc/webhook-service 8001:8001
kubectl port-forward svc/ai-service 8000:8000
# Then: curl http://localhost:8001/webhook/github
```

### Check Status
```bash
kubectl get pods -A
kubectl describe pod <pod-id>
kubectl get svc -A
```
